## 7. Mobile Responsive Layouts

### 7.1 Breakpoints

- **Mobile**: 320px - 640px
- **Tablet**: 641px - 1024px
- **Desktop**: 1025px - 1440px
- **Large Desktop**: 1441px+

### 7.2 Responsive Behavior

**Mobile (< 640px)**:
- Sidebar collapses to hamburger menu
- Single column layouts
- Stacked navigation
- Full-width cards
- Bottom navigation bar for quick access

**Tablet (641px - 1024px)**:
- Collapsible sidebar (drawer)
- 2-column grids where appropriate
- Compact navigation
- Touch-optimized buttons (min 44px)

**Desktop (1025px+)**:
- Full sidebar navigation
- Multi-column layouts
- Hover states enabled
- Keyboard navigation support

---

## Component Specifications

### Order Card Component
- **Width**: Auto (grid item)
- **Padding**: 16px
- **Border Radius**: 8px
- **Shadow**: Subtle elevation
- **Hover**: Slight lift and shadow increase
- **Status Badge**: Color-coded (green/blue/orange)

### Product Card Component
- **Image Aspect Ratio**: 16:9
- **Padding**: 12px
- **Border Radius**: 12px
- **Action Buttons**: Icon buttons, 40px size

### Stats Card Component
- **Height**: 120px
- **Padding**: 24px
- **Icon Size**: 32px
- **Value Font**: 28px, bold
- **Label Font**: 14px, medium

## Notes

- All wireframes are based on a 12-column grid system
- Consistent spacing using 4px base unit
- Touch targets minimum 44x44px for mobile
- All interactive elements have hover/focus states
- Loading states included for async operations
- Error states designed for user feedback
